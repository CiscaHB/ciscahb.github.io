Relational vs NoSQL Normalisation INNER JOIN LEFT JOIN Data Integrity Table Relationships

Infographic created with NotebookLM — database fundamentals, SQL vs NoSQL architecture, normalisation, and JOIN types.

Key takeaways
  • Relational databases use predefined tables for high data integrity — ideal for structured records like payments and student data.
  • NoSQL databases use flexible JSON or key-value pairs, suited to unstructured data like website activity logs.
  • Normalisation splits data across specialised tables, reducing redundancy and improving query speed.
  • INNER JOIN returns only matching records across both tables; LEFT JOIN includes all rows from the primary table even where no match exists.

Cities in Europe — SQL Query with JOIN

A practical MySQL query joining a country table to a city table to extract every European capital city, filtered by continent and sorted alphabetically by country.

SELECT JOIN … ON WHERE ORDER BY Aliasing (AS) Multi-table Query

MySQL Workbench — query #14 from the Data Bootcamp SQL repository. Returns capital cities across all European countries, ordered by country name.

What this query does
  • JOIN: Links the country table to the city table on the capital city ID, combining data from two sources in one result.
  • WHERE: Filters rows to only European countries, reducing the dataset to the relevant continent.
  • Aliasing: Renames columns (c.Name as CapitalCity) for clarity in the result grid.
  • ORDER BY: Sorts results alphabetically by country name, making the output easy to scan.
